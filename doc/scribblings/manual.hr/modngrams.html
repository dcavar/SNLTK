4 ngrams - N-gram modeli

Damir Ćavar

 (require nltk/ngrams)
nltk/ngrams modul daje postupak za generiranje i procesiranje n-gram modela iz dane liste tokena. Lista tokena može biti slijed bilo koje vrste. Za lingvističke analize i procesiranje tokeni su najčešće stringovi koji predstavljaju riječ, ili su samo nizovi znakova.

(token-sequence->ngrams seq) → hashtable?
  seq : (or/c list? vector? string?)
Vraća strukturu bigram podataka kao hashtable u kojoj je bigram ključ, a apsolutni brojem vrijednost. Parametar seq mora biti sekvencionalan, odnosno lista, vektor simbola ili string. Budući da je string definiran kao niz slova, konačni rezultirajući bigram model se sastoji od dva niza simbola. Broj simbola u konačnom n-gram modelu je određen implicitnim parametrom n, koji mora biti pozitivan, veći od 0 i po već zadanom standardu je 2.

(token-sequence->ngrams seq n) → hashtable?
  seq : (or/c list? vector? string?)
  n : (>/c 0)
Vraća n-gram strukturu podataka kao hashtable, u kojoj je n-gram ključ, a apsolutni brojem vrijednost. Parametar seq mora biti sekvencijalnog tipa, što znači da mora biti lista, vektor simbola ili string - što je interpretirano kao niz znakova. Broj znakova u rezultirajućem n-gram modelu je određen parametrom n, koji mora biti pozitivan i veći od 0.

(token-sequence->ngrams seq n ngrams) → hashtable?
  seq : (or/c list? vector? string?)
  n : (>/c 0)
  ngrams : hashtable?
Vraća n-gram strukturu podataka u obliku hashtable, u kojoj je n-gram ključ, a apsolutni brojem vrijednost. seq parametar mora biti sekvencijalnog tipa, odnosno lista, vektor sa oznakama/simbolima ili string, koji se interpretira kao niz znakova. broj simbola/oznaka u dobivenom n-gram modelu je određen parametrom n, koji mora biti pozitivan i veći od 0.

(ngrams->html-table ngrams    
  column-titles    
  sorted    
  sort-field) → string?
  ngrams : hashtable?
  column-titles : list?
  sorted : boolean?
  sort-field : symbol?
Vraća string koji sadrži HTML kod koji reprezentira n-grame i njihove frevkvencije kako su pohranjene u strukturi podataka hashtable s varijablom ngrams. Opcionalni parametar column-titles je lista (list) stringova (strings) sa stupcima ili oznakama zaglavlja za dobivenu tablicu. Opcionalna sorted oznaka je podešena na #t, što rezultira sortiranim outputom (izlazom), koristeći strategiju "od većeg prema manjem" za brojeve, i "od manjeg prema većem" za stringove. opcionalni parametar sort-field je ili simbol 'frequency ili nešto drugo. Ako sort-field ima vrijendost 'frequencyonda će dobivena tablica biti sortirana od veće prema manjoj vrijednosti frekvencije n-grama koje sadržava. Ako ne, onda će tablica biti sortirana od manje vrijednosti prema većoj s obzirom na same n-grame (n-grami se onda tretiraju kao stringovi).

Svaki red HTML tablice sadrži listu ćelije u kojima su svi tokeni iz n-grama kao i njihove odgovarajuće vrijendosti. Krajnji HTML kod tabele omogućava CSS, Javascript ili style hook za podešavanje opcija preko class="table-style" atributa na table oznaku.

(ngrams->bigrams ngrams) → hashtable?
  ngrams : hashtable?
Vraća novu strukturu podataka hashtable, gdje su bigrami ključ, a frekvencije vrijednosti. Ako je dužina n-grama u parametru ngrams manja od 2, ili ako je ngrams struktura podataka prazna onda se vraća prazna bigrams hashtable. Frekvencije bigrama u tako dobivenoj strukturi podataka ne odražava realne frekvencije tih n-grama u originalnom izvoru teksta ili korpusu.

(filter-ngrams ngrams filter-tokens remove) → hashtable?
  ngrams : hashtable?
  filter-tokens : list?
  remove : boolean?
Vraća novu strukturu podataka u obliku hash-tabele hashtable, gdje su n-grami ključ, a frekvencije vrijednost. Ako je opcionalna remove oznaka zadana kao #t, vraćena hashtable neće sadržavati nijedan n-gram koji sarži element koji je member obavezne filter-tokens liste. Ako je opcionalna oznaka remove zadana kao #f, vraćena hashtable će sadržavati samo n-grame koji sadrže element koji je član member obavezne filter-tokens liste.

(filter-ngrams! ngrams filter-tokens remove) → hashtable?
  ngrams : hashtable?
  filter-tokens : list?
  remove : boolean?
Vraća potpuno istu strukturu podataka oblika hashtable koja je određena parametrom ngrams, gdje je n-gram ključ, a odgovarajuće frekvencije n-grama vrijednost. Slično filter-ngrams, ako je remove oznaka postavljena kao #t svi n-grami će biti uklonjeni iz ngrams hashtable ako je samo i jedan token member liste filter-tokens. Ako je pak "remove" oznaka postavljena kao #f u rezultuirajućoj hastable će ostati samo n-grami koji sadrže token koji je dio member liste filter-ngrams. struktura podataka ngrams je promijenjena.
